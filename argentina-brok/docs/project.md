## Архитектура проекта

Данный проект построен на основе архитектуры `FSD - Feature Slice Design`.

FSD - это методология проектирования фронтенд-приложений, основанная на:
1. **Слоистой структуре** - четкое разделение кода по уровням абстракции
2. **Бизнес-ориентированности** - организация кода вокруг функциональности продукта
3. **Изоляции модулей** - минимизация cross-импортов между слоями

```
app/                # Маршрутизация Next.js (App Router)
  view/               # UI-представления страниц (адаптеры для страниц)
  widgets/            # Независимые составные компоненты
  i18n/               # Локализация и интернационализация
  features/           # Бизнес-фичи продукта
  entities/           # Бизнес-сущности домена
  shared/             # Общие инфраструктурные модули
public/               # Статические активы
```

Документация на архитектуру - [FSD](https://nodejs.org/)

1) `app/` — UI-представления страниц.
Отвечает за визуальное оформление конкретных страниц.

2) `widgets/` — UI-блоки, ориентированные на сценарии.
    Самостоятельные блоки, которые агрегируют features, entities, shared/ui.
    Используются на разных страницах, могут быть повторно применимы.

3) `features/` — Бизнес-фичи.
    Самый важный слой: каждая фича — это завершённый пользовательский сценарий.
    Состоит из ui/, model/, lib/ и может использовать entities и shared.
    Главные фичи проекта - графики и формы.

4) `entities/` — Базовые доменные сущности.
    Представляют собой фундаментальные объекты бизнес-домена.
    Могут содержать ui, model, api, types.

5) `shared/` — Общие утилиты и компоненты.
    Общая инфраструктура, не зависящая от домена.

    Содержание:

    - `ui/` — дизайн-система, атомарные компоненты, которые используются в дальнейшем по всему приложению

    - `api/` — общие HTTP-клиенты, axios-interceptor

    - `config/` — конфигурации - настройка локальных шрифтов проекта

    - `types/` — глобальные типы, которые используются по всему приложению

    - `lib/`, `hooks/` и `helpers` - хелперы, форматтеры, функции, вспомогательные хуки

    - `styles/` - глобальные стили

    - `model/` и `constants/` - переиспользуемые константы

6) `i18n/` — Локализация.
    Все файлы интернационализации и настройки.

# Правила верстки

## Система отступов: Зачем нужен миксин `responsive`

Когда нужно задать отступы или другие CSS-свойства так, чтобы они выглядели хорошо на всех экранах — от мобильных до больших мониторов — используем миксин `responsive`. Он позволяет задать значения один раз, а дальше всё само масштабируется в зависимости от ширины экрана.

## Глобальная настройка

> В проекте глобально задан базовый размер шрифта:

```javaScript
html {
   	@media (max-width: 1440px) {
		font-size: 1vw;
	}

	@media (min-width: 1441px) {
		font-size: 14.4px;
	}
}
```

> Это обеспечивает масштабирование элементов в зависимости от ширины окна браузера. Таким образом, значения в rem становятся адаптивными до 1440px а от 1441px остается неизменяемый размер отступов.

## Структура миксина

```javaScript
@define-mixin responsive $property, $mobile-size, $tablet-size, $desktop-size {
	@media (max-width: 767px) {
		$(property): calc($(mobile-size)rem / 3.75); // Для мобилок
	}

	@media (min-width: 768px) and (max-width: 1440px) {
		$(property): calc($(tablet-size)rem / 14.4); // для десктопа
	}

	@media (min-width: 1441px) {
		$(property): calc($(desktop-size)rem / 14.4); // для больших экранов
	}
}

```

## Миксин принимает 3 параметра:

● `$property` — нужное CSS-свойство (например, `max-width`, `padding-block-start`, `gap` и т.п.)

● `$mobile-size` — отступ в пикселях для мобильной версии (до `767px`)

● `$tablet-size` — отступ для десктопов (от `768px` до `1440px`)

● `$desktop-size` — отступ для десктопов (от `1441px` и выше)

## Как работает миксин `responsive`

● Значения переводятся из пикселей в rem

● Делятся на:

1. `3.75` — это ширина экрана на макете для мобильных (в rem, `375px / 100`)

2. `14.4` — это ширина экрана на макете для десктопов (`1440px / 100`)

3. от `1441px` - это ширина экрана, где размеры перестают быть адаптивными под ширину экрана, и остаются неизменными

● А дальше уже `font-size: 1vw` всё адаптирует под ширину экрана

### Пример использования

```javaScript
@mixin responsive padding-block-start, 60, 120, 192;
```

### Этот вызов задаёт:

```javaScript
 padding-block-start: calc(60px / 3.75); // → ≈ 32rem - на мобилках
 padding-block-start: calc(120px / 14.4); // → ≈ 13,4rem - на десктопе
 padding-block-start: 192px; //→ остается 192px от 1441px
```

И всё это будет адаптивно меняться при изменении ширины окна — без медиазапросов.

## Когда и как использовать

● Всегда, когда задаёте внешние или внутренние отступы

● Значения указывайте в пикселях, как обычно — миксин сам всё переведёт

● Проверяйте адаптивность на экранах около `767px` — там переключаются условия


# Система шрифтов

В проекте используется адаптивная система шрифтов, основанная на брейкпоинтах.

## Миксины для шрифтов

Каждый миксин включает в себя параметры шрифта: `размер`, `межстрочный интервал`, `межбуквенное расстояние` и другие свойства. Эти миксины можно применять к любым элементам.

### Примеры миксинов

```css
@define-mixin font-h1 {
    font-size: var(--h1);
    line-height: var(--h1-line-height);
}

@define-mixin font-h2 {
    font-size: var(--h2);
    line-height: var(--h2-line-height);
    letter-spacing: var(--h2-letter-spacing);
}

@define-mixin font-h3 {
    font-size: var(--h3);
    line-height: var(--h3-line-height);
}
```

> Все значения задаются через `CSS-переменные`, которые указываются в `px`.

## Пример переменных

```css
@media (max-width: 767px) {
    --h1: 32px;
    --h2: 24px;
    --h3: 20px;
    --h4: 18px;
    --h5: 16px;
    --h6: 16px;
    --text20: 16px;
    --text18: 16px;
    --text16: 14px;
    --text14: 14px;
    --text12: 12px;
}
```

## Адаптивность через `media-запросы`

Если в макетах представлены несколько брейкпоинтов, следует использовать media-запросы для переопределения переменных.

### Пример для экранов шириной от `1440px`

```css
@media (min-width: 1440px) {
    --h1: 64px;
    --h2: 40px;
    --h3: 32px;
    --h4: 24px;
    --h5: 20px;
    --h6: 18px;
    --text20: 20px;
    --text18: 18px;
    --text16: 16px;
    --text14: 14px;
    --text12: 12px;
}
```

## Поддержка локалей

В файле `global.css` можно задать разные значения переменных для конкретных языков с помощью селектора `:lang(...)`.

### Примеры:

```css
:root:lang(hy) { /* армянский */
    @media (max-width: 767px) {
        --h1: 32px;
        --h2: 24px;
        --h3: 20px;
        --h4: 18px;
    }
}

:root:lang(ru) { ... } /* русский */
:root:lang(en) { ... } /* английский */
```

## Применение миксинов

```css
.title {
    @mixin font-h1;
}

.description {
    @mixin font-h4;
}
```

## Итого

* Все размеры задаются в `px`.
* Используются `CSS-переменные` для управления размерами.
* Миксины помогают избежать дублирования кода.
* Адаптивность достигается через переопределение переменных в `media-запросах`.
* Поддерживается локализация через `:lang(...)`.
